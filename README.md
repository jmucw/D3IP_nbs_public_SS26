# D3IP - Data Driven Decisions in Practice: Urban Analytics

## Getting Started

Follow these steps to set up your coding environment and run the notebooks.

### 1. Create a GitHub Account

If you don't have one yet, go to [https://github.com](https://github.com) and sign up for a free account.

### 2. Open the Repository in GitHub Codespaces

1. Make sure you are logged into GitHub.
2. Navigate to this repository.
3. Click the green **"<> Code"** button near the top right of the page, above the file list.
4. In the dropdown that appears, select the **"Codespaces"** tab (next to the "Local" tab).
5. Click **"Create codespace on main"**.

This will open a VS Code editor in your browser with all required packages pre-installed. The setup may take a couple of minutes on first launch.

### 3. Open and Run a Notebook

1. In the file explorer on the left, click on a notebook file (e.g., `1_data_handling.ipynb`).
2. The notebook will open in the editor.
3. When prompted to select a kernel, choose **Python 3.12.13**. **Important: There are two Python kernels available (3.12 and 3.13). You must select Python 3.12.13 — the 3.13.5 kernel does not have the required packages installed and will not work.**
4. You can now run cells by clicking the **play button** next to each cell, or by pressing **Shift + Enter**.

### 4. Saving Your Work

**Note:** Free GitHub accounts include a limited number of Codespace hours per month. Make sure to stop your Codespace when you are not using it. You can do this by clicking on "Codespaces" at the bottom left of the VS Code window and selecting "Stop Current Codespace".

If you only want to run the notebooks without saving changes, you can work directly in the Codespace as described above. However, if you want to save your changes (e.g., your solutions to the exercises), you need to create a personal copy of the repository first:

1. Go to this repository.
2. Click the **"Fork"** button in the top right corner of the page.
3. On the next screen, click **"Create fork"**. This creates a copy under your own GitHub account.
4. Open the Codespace from **your fork** (not the original repository) by following steps 2 and 3 above.

Any changes you make in a Codespace launched from your fork will be saved to your personal copy. You can return to your Codespace at any time via [https://github.com/codespaces](https://github.com/codespaces).
